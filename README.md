# WFInfo 中文版

WFInfo 是 Warframe 的配套应用程序，基于英文版开发。

WFInfo 旨在为所有裂缝任务奖励提供快速的白金和杜卡德价格查询，帮助您轻松选择最佳奖励。

## 工作原理

WFInfo 通过截取游戏窗口截图，裁剪出部件文本，然后将其传递给光学字符识别引擎（OCR）进行识别。OCR 引擎会返回识别到的文本，我们从中提取部件名称，然后在覆盖层或独立窗口中显示每个部件的统计数据。

1. 下载最新版本并解压
2. 以管理员身份运行 WFInfo.exe，等待初始加载完成（数据库 + OCR 数据）
3. 进入 Warframe 游戏，将显示模式设置为**无边框全屏**，并在界面设置中开启**物品标签**
4. 在裂缝任务奖励界面按 **Print Screen** 热键显示价格，或使用自动功能等待自动触发

## 功能特性

### 丰富的部件信息

当 WFInfo 显示部件信息时（通过覆盖层或独立窗口），会显示以下数据：

- **拥有数量**（基于装备窗口）
- **已入库标签**（如果部件已入库）
- **部件名称**（用于确认没有识别错误）
- **白金价值**（基于 warframe.market 平均价格）
- **杜卡德价值**
- **销量**（过去 48 小时内在 warframe.market 的销量）

### 自动模式

启用后，程序会监听调试日志（EE.log），等待显示奖励的消息出现。当前触发关键词是 "Got rewards"。检测到该消息后，会自动触发显示功能，在覆盖层/窗口中显示奖励信息。

如果在 5 秒内无法检测到奖励界面，则假定发生了错误。

如果没有显示，您可以按热键强制激活。

### 剪贴板复制

启用后，WFInfo 会将白金价格和部件名称复制到剪贴板，您可以直接粘贴到 Warframe 聊天中与队友分享。

### 自动更新

在 WFInfo 初始加载期间，会查询我们的网站检查是否有可用更新。如果发现更新，会显示提示，您可以让它自动下载并更新 WFInfo。

### 遗物面板

遗物面板允许您查看所有遗物及其统计数据。我们显示基本信息，如遗物是否已入库、遗物奖励及其稀有度，以及每个遗物的三个复杂统计数据：

- 完整遗物奖励的平均白金价格（基于掉落概率）
- 光辉遗物奖励的平均白金价格
- 两者之间的差值（可用于找出精炼后能获得更多白金的遗物）

### 装备面板

装备窗口允许您查看每个 Prime 装备及其部件。显示每个部件在 warframe.market 上的当前价格，以及购买整套装备的费用。

如果您在此标记物品，它会在裂缝任务奖励界面显示。

### Snap-it 快速扫描

查看您的 Prime 库存时，可以扫描它以查看每个物品价值多少白金、杜卡德或白金/杜卡德比率，对 Baro 的杜卡德兑换非常有用，或快速查看可以在 WFM 上出售的物品。

扫描的物品列表甚至可以导出为 CSV 文件。

### Warframe.Market 连接

- 通过检测游戏是否运行或检测 AFK 状态，自动管理您的 Warframe.Market 在线状态
- 按两个键即可搜索特定物品，甚至可以在不退出游戏的情况下将其添加到出售列表
- 通过点击"确认上架"按钮，在任务结束时列出您选择的 Prime 奖励
- 如果您喜欢这个软件，甚至可以一键在开发者资料上发布评论！

## 设置选项

### 显示设置

- **显示模式**：覆盖层 / 突出显示 / 窗口
- **主题**：自动检测 / 亮色 / 暗色

### 自动功能

- **自动**：启用日志监听，自动触发识图
- **自动列表**：任务结束时自动创建 Warframe.Market 出售列表
- **自动CSV**：任务结束时自动保存奖励数据到 CSV 文件
- **自动计数**：任务结束时自动增加物品拥有数量

### 热键设置

- **激活热键**：默认 Print Screen

## 常见问题

### 自动功能不工作？

1. 确保以**管理员身份**运行 WFInfo
2. 确保游戏显示模式设置为**无边框全屏**
3. 检查 debug.log 中是否有 "LogCapture initialized successfully"
4. 如果 DBWIN 事件被占用，尝试重启电脑

### 识别错误？

1. 确保游戏界面设置中开启了**物品标签**
2. 确保游戏分辨率足够高（建议 1920x1080 或更高）
3. 尝试手动按热键触发
